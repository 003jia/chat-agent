# 聊天智能体项目总览

## 项目定位

这是一个本地优先的聊天智能体工作台，核心目标是验证三件事：

- 智能体可以先配置角色、行为和模型，再进入对话。
- 对话过程中的稳定事实和长期偏好可以进入候选记忆区，由用户审核后写入长期记忆。
- 后续对话能按相关性和重要等级加载长期记忆，而不是把所有历史无差别塞进 prompt。

## 当前版本完成内容

### 产品功能

- 桌面端三栏工作台：左侧角色/模型配置，中间聊天，右侧长期记忆上下文。
- 移动端单页聊天和设置页。
- 多角色预设和多会话管理。
- OpenAI-compatible、OpenAI、DeepSeek、Anthropic 四类模型配置。
- 流式对话输出和 Markdown 消息渲染。
- 联网搜索模式，搜索内容作为不可信上下文注入。
- 长期记忆候选审核、提交、拒绝、编辑和本地整理。

### 记忆结构

- `index.json` 保存结构化记忆。
- `raw/YYYY-MM-DD.md` 作为 append-only 审计日志。
- `memory.md` 作为自动生成的长期记忆摘要。
- 记忆检索使用中文友好分词、常驻高优先级偏好桶、关键词命中桶和时间衰减。
- 候选记忆支持 `add`、`update`、`noop` 语义，避免重复偏好和矛盾偏好长期堆积。

### 安全与可靠性

- 本地 API 默认绑定 `127.0.0.1`。
- CORS 限制为本地白名单。
- 写接口、聊天接口和模型测试接口需要管理令牌。
- 模型调用有超时，聊天接口有限流和长度校验。
- API Key 可通过环境变量覆盖本地配置。
- `data/`、`.env*`、`.workbuddy/` 不提交。

### 工程化

- 前端拆分为工作台、聊天、设置、记忆、抽屉和通用 UI 组件。
- 后端拆分为配置、模型、搜索、记忆、会话、错误和日志模块。
- 已接入 Vitest 测试。
- 已添加 GitHub Actions CI。
- 已添加 PR 模板。

## 验证方式

```bash
npm test
npm run build
npm run qa:screenshots
```

## 不上传内容

以下内容只保留在本地：

```text
data/
.env
.env.*
.workbuddy/
qa-screenshots/
```

其中 `data/config/models.json` 可能包含本地 API Key，不能提交到 GitHub。

## 后续方向

- 模型版 memory organize：合并重复项、消解矛盾、淘汰过期项，并提供预览确认。
- 正式搜索 API：保留 HTML 搜索作为 fallback。
- Embedding 语义检索：本地缓存向量，失败时降级关键词检索。
- 更完整的记忆审计视图：展示每条回答引用了哪些具体记忆。
