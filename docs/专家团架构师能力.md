# 专家团架构师能力

## 来源

本能力根据 2026-07-23 整理的 `Comate 专家团编写规范` 接入聊天智能体，运行标识为：

```text
expert-team-authoring
```

对应内置角色：

```text
role-expert-team-author
```

## 能做什么

- 根据团队目标生成 Team 插件目录、Manifest、settings、Lead 和 Member 定义。
- 把 Skill 编排的多 Agent 插件迁移成唯一 Lead 负责状态的 Team 插件。
- 审查 Agent ID、资源路径、路由、Workflow DAG、权限确认和完成证据。
- 为每个工作流节点定义 owner、depends_on、input、output、acceptance、failure_route 和 max_retries。
- 输出任务信封、结果信封、完成门禁与未验证假设。

## 核心协议

```text
Team Plugin
├── Manifest
├── Lead Agent
├── Member Agent Definitions
├── Runtime Subagent Sessions
├── Workflow DAG
├── Shared Artifacts
└── Permission Policy
```

Lead 只负责路由、状态、依赖、重试、验收和汇总。Member 每人只承担一个不重叠的结果责任。先设计路由和 DAG，再编写人格描述。

显式状态：

```text
pending
running
needs_user
blocked
failed
completed
```

编码交付必须提供修改文件、实际命令、退出码、测试结果、未验证项和残余风险。计划、伪代码和静态审查不能冒充真实执行证据。

## 权限边界

删除、覆盖、发布、生产变更、付费资源、凭据、对外发送和不可逆操作必须由用户确认。提示词中的禁止不等于权限隔离。

## 当前产品边界

聊天智能体当前接入的是“专家团设计与审查能力”，不是多 Agent 运行时。角色可以生成和审查完整专家团协议，但不能声称已经真实启动成员会话。只有后续接入可验证的 Subagent Runtime 后，才能报告成员执行结果。

## 使用方式

1. 打开设置。
2. 在角色预设中选择“专家团架构师”。
3. 点击快捷任务，或直接输入团队目标、现有插件目录和待解决问题。

内置角色不能删除，但可以调整名称、描述、头像、语调和开场白。能力协议由 `capabilityIds` 保持，不会因修改显示信息而丢失。
