# 增量 PRD：Memory Agent Workbench 陪聊化改造

## 1. 项目信息

| 字段 | 内容 |
|------|------|
| Language | 中文 |
| Programming Language | React 19 + TypeScript + Vite 7（前端）；Node.js + Express 5（后端） |
| Project Name | memory_agent_companion_refresh |
| 原始需求复述 | 将现有「Memory Agent Workbench」从三栏调试工作台改造为以聊天为中心的温暖陪聊产品，覆盖界面重塑、聊天体验、陪伴感功能、假功能清理四个方向，同时控制本轮范围不膨胀。 |

## 2. 产品定义

### 2.1 Product Goals

1. **从工作台到聊天主界面**：将三栏布局压缩为以对话为核心的单主视图，设置与记忆管理退入抽屉/弹层，降低工具感与认知负荷。
2. **人格化陪伴体验**：通过名字、头像、性格、主动问候、记忆自然引用，让用户感受到与"一位持续在场的 AI 伙伴"对话，而非与工具交互。
3. **修复信任损伤**：清理当前对外可见的假功能（摘要模板、无效按钮、死开关），确保每个可点击入口都有真实、可预期的行为。

### 2.2 User Stories

- **US-1（初次用户）**：作为第一次打开应用的用户，我希望默认看到干净的聊天界面，而不是三栏工作台，以便立刻开始对话。
- **US-2（情感陪伴用户）**：作为把 AI 当陪伴对象的用户，我希望给 TA 起名字、选头像、写性格，让 TA 更像一个具体的人。
- **US-3（回归用户）**：作为隔几天再回来的用户，我希望 AI 在回复里自然提到"我记得你说过……"，而不是显示"引用 2 条记忆"，以便感受到被记住。
- **US-4（对话控制用户）**：作为日常聊天用户，我希望每条助手消息都能重新生成或一键复制，并看到发送时间，以便随时修正和回顾。
- **US-5（沉淀型用户）**：作为想把对话价值留下来的用户，我希望点击"生成摘要"后得到真正基于对话内容的摘要，而不是模板废话，再决定是否沉淀为长期记忆。

## 3. 技术规范

### 3.1 Requirements Pool

方向标签：
- `[界面]` 界面风格重塑
- `[聊天]` 聊天体验细节
- `[陪伴]` 陪伴感功能
- `[清理]` 清理假功能 / 无效入口

#### P0 — 本轮必做

| 编号 | 方向 | 需求 | 验收标准（可测试） |
|------|------|------|-------------------|
| P0-1 | `[界面]` | 默认聊天为主视图，设置栏与记忆面板收入抽屉/弹层 | 桌面端与移动端均默认只展示聊天区；设置/记忆入口收敛到顶部操作栏或抽屉触发按钮；现有左侧 `AgentSidebar` 与右侧 `MemoryPanel` 不再常驻显示。 |
| P0-2 | `[界面]` | 顶部标题栏人格化 | 聊天顶部显示伴侣名字、头像/颜色标识、在线状态点；弱化"Memory Agent Workbench"品牌露出，会话标题旁的技术状态文案折叠或移除。 |
| P0-3 | `[陪伴]` | 人设自定义扩展 | 在现有 `agentConfig.name` / `roleTitle` / `roleDescription` 基础上新增：`avatar`（emoji 或纯色块标识）、`accentColor`（主题色）、`personalityTone`（性格口吻描述）；设置抽屉提供编辑入口；数据持久化到 `data/config/roles.json`。 |
| P0-4 | `[陪伴]` | 主动开场白 | 新建会话或空会话（messages 为空）时，自动显示一条预定义问候语；问候语可从人设 `greeting` 字段读取，未设置则使用默认文案。 |
| P0-5 | `[陪伴]` | 记忆自然外露 | 当回复引用记忆时，system prompt 要求模型以自然句式（如"我记得你说过……"）融入正文；前端将"引用 N 条记忆"技术标签改为可展开的记忆脚注，默认折叠。 |

| P0-8 | `[聊天]` | 流式打字动效 | 流式回复期间显示平滑闪烁光标或脉冲头像，替代当前空白占位；滚动保持自然跟随。 |
| P0-9 | `[清理]` | 会话摘要真接入模型 | 按现有 PRD `docs/PRD-会话摘要真接入AI.md` 实现后端 `POST /api/conversations/:id/summary` 接口，替换前端 `generateSummary()` 的模板拼接假实现。 |
| P0-10 | `[清理]` | 隐藏无实际功能的入口 | 在实现前隐藏 composer 中的语音（Mic）、附件（Paperclip）按钮；联网搜索按钮若确认后端无实际功能则一并隐藏，若已实现则移入工具抽屉/菜单；暂时隐藏或禁用 en 语言切换，待确认需要国际化后再开放。 |

### 3.2 UI 设计要点

#### 布局结构

- **桌面端**：
  - 主区域：单一聊天面板，居中或撑满剩余空间，最大宽度建议 720–800px。
  - 顶部栏：左侧返回/会话入口（抽屉触发）、中间伴侣头像+名字+状态、右侧设置/记忆/搜索图标。
  - 设置抽屉：从左侧滑出，包含会话列表、角色预设、人设编辑、行为开关、温度、模型配置。
  - 记忆抽屉/弹层：从右侧滑出或作为可拖拽侧边面板，展示已加载记忆与候选记忆审核。
  - 底部输入区：简化 composer，移除 Mic/Paperclip；保留发送按钮与占位提示文案。

- **移动端**：
  - 移除"手机外壳"装饰框，改为纯响应式 Web 视图。
  - 同样以聊天为主，设置通过顶部按钮进入全屏/抽屉设置页。
  - 底部 composer 固定，避免键盘拉起时遮挡输入框。

#### 视觉基调

- 整体从"冷白工具界面"转向"温暖聊天室"：
  - 背景：低饱和奶油/浅灰，保留轻微玻璃质感但降低对比度。
  - 用户气泡：柔和主色（可随 `accentColor` 变化），圆角 16–20px。
  - 助手气泡：白色/浅灰，带极淡阴影，圆角 16–20px。
  - 字体：正文保持可读性，减少小号技术标签；名字与状态使用中等字重。
- 头像：默认使用 emoji 或双色块，避免复杂图标，保持亲和力。

#### 关键交互

- **抽屉开关**：顶部设置/记忆按钮点击滑出对应抽屉；再次点击或点击遮罩关闭。
- **开场白**：空会话自动以 assistant 身份插入一条问候气泡，不经过用户发送，但标记为 `isGreeting: true`。
- **记忆脚注**：助手消息底部以小字显示"引用了 X 条记忆"，点击展开具体条目；高优先级记忆若已自然融入正文，不再重复列出。
- **重新生成**：点击后当前 assistant 消息进入 loading 态，流式替换内容；失败保留原消息并提示错误。
- **复制**：点击后 toast 提示"已复制"，支持 Markdown 原文复制。

### 3.3 Open Questions

| # | 问题 | 我的推荐 | 需要确认 |
|---|------|---------|---------|
| Q1 | 联网搜索当前是否真实可用？ | 后端 `search.mjs` 与 `api.webSearch` 均存在，疑似已实现；建议 P0 先将其从 composer 主按钮区移入工具抽屉，不再作为默认陪聊入口。 | 是否确认后端搜索链路可跑通？ |

| Q4 | en 语言开关处理？ | 当前 `src/i18n.ts` 中 en 文案已较完整，但产品定位中文陪聊；P0 建议隐藏切换器减少噪音，P1 再评估国际化。 | 是否必须保留双语切换？ |

| Q7 | 左侧会话列表是否完全隐藏？ | P0 收入抽屉顶部，不再常驻；但保留快捷新建会话按钮在聊天顶部。 | 是否希望桌面端保留一个极窄的会话条？ |
